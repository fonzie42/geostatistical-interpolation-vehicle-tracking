# Figures

This directory holds the final thesis figures (PNG). Each was regenerated and verified
from the committed inputs in `data/`, `results/`, and `logs/`. The generating scripts live
in `preprocessing/` and `analysis/`.

**Run every command from the repository root** with the Python environment from
`requirements.txt` active (the scripts use paths relative to the repo root and import the
`preprocessing` package). In the examples below, `python` is the interpreter from that
environment.

| Fig | Script | Inputs | Command | Output written by script | Copy here |
|-----|--------|--------|---------|---------------------------|-----------|
| 5.1 logit explainer | `analysis/logit_explainer_fig.py` | `analysis/predictions_delta005.csv` | `python analysis/logit_explainer_fig.py` | `analysis/logit_explainer.png` | `logit_explainer.png` |

Notes:

- **Committed vs. regenerated.** This `figures/` directory holds exactly the images used in
  the thesis `.tex`, committed once each. The "Output written by script" column points to
  where each script *writes* when you run it (`results/` or `analysis/`); those runtime
  outputs are git-ignored to avoid shipping duplicate or intermediate images. To verify a
  figure, run its script and compare the regenerated file against the copy here.
- **Fig 5.8 naming.** The script writes `reliability_logit.png`; the thesis includes it as
  `reliability_pit.png`. The copy in this directory uses the thesis name.
- **Fig 4.1.** Regenerated by `generate_observations_map.py`, which calls
  `plot_observed_data` on `data/delta_0.05.csv` with the RS boundary and no basemap.
  The regenerated PNG is byte-identical to the committed copy here.
- **`generate_native_panels.py`** regenerates all three native panels (5.6, 5.7, 5.9) in a
  single run and also writes matching `.svg` files into `results/`.
- **`generate_logit_utm_maps.py`** regenerates all three logit maps (5.3, 5.4, 5.5) in a
  single run and also writes `.svg` files into `results/`.
- **`preprocessing/task8_events_per_cell.py`** (events-per-cell histogram, supporting
  figure) reads the **1 GB raw parquet**, which is not committed. Obtain the parquet first
  (see `data/README.md`) before running it.
